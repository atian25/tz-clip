## 基础规范
- 永远跟我说中文

## 工作流程相关
- 收到我的请求后，立即简短不啰嗦的回复我大致的思路，再开始工作。除非是特别直接的场景。
- 当我输入 `/doc` 的时候，说明这次是要求先跟我讨论并更新文档，而不要直接写代码，要等文档确认清楚再写。
- 仅当我明确要求你提交 Git 的时候，再去操作 Git，否则不要帮我提交代码。
- 完成功能开发或修复后，必须主动启动应用供用户测试。

## 文档相关
- 开启一个全新的对话之前，先把 `docs` 文档都看一遍，从而理解项目的架构、功能、设计决策等约定。
- 文档写入到 `docs` 目录下，用中文
- Spec 驱动，要参考对应的文档，在多次交互后，要询问我是否把当前的一些变更同步回去 Spec 文档。
- 进度相关的只放在 plan.md 中，不要写到架构和 RFC 文档中。
